Pathologies the audit flags

None of these move precision@K cleanly. All are visible in the shape of your retrieval distribution before they surface as user complaints.

Score miscalibration
Rank order stays right while score values drift as the corpus grows. Precision@K is blind to it.
Rank inversion
Top-K contains all relevant docs but in reverse order. LLM reads top-down, hallucinates.
Redundancy attacks
Near-duplicate sources evict distinct ones from top-K. Answer is confidently incomplete.
Bimodal retrieval
Two content clusters both score high. Wrong cluster wins top-K. Confidently wrong answer.
Contamination drift
Off-topic chunks slowly accumulate in top-K. Mean alignment drops until complaints arrive.
Tool fixation / agent loops
Per-tool-call relevance distribution collapses. Loop detectable several turns before production breaks.

How this relates to the open-source sensors

The five Bell Tuning sensors are MIT-licensed and installable via npx contrarianai-context-inspector --install-mcp. They are free. If you want to run them yourself, you do not need this audit.

The audit exists for teams that want the analysis done for them in 48 hours with a defensible report attached. You're paying for the turnaround, the pathology interpretation, and the prioritized fix list - not the tools themselves.
