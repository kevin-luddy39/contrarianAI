What is Bell Tuning?

Bell Tuning is the practice of treating an AI context window as a measurable distribution — and tuning your workflow against the shape of that distribution rather than against the output it eventually produces.

Bell ShapeWhat it means
Tight, right-shiftedContext is on-domain and consistent. Healthy.
Wider, drifting leftContamination, summary loss, or topic drift entering. Tune now.
Flat near zeroOriginal content gone. System still answering, but on noise.

You don’t tune by listening to the output. You tune by watching the bell. Context Inspector is the instrument.

The Research

"We found that context degradation follows predictable statistical patterns. The bell curve reveals them early — before output evaluation catches up."

The white paper documents a controlled experiment: fairy tales fed through an AI system with a constrained context window, progressive contamination, and forced summarization. The bell curve caught the decay 3 steps before the output failed.

From White Paper to Tool

Every metric in Context Inspector was designed to surface the specific signals the research identified:

  • Fixed domain reference — the paper showed that scoring against a shifting vocabulary masks drift. The tool freezes domain terms at baseline.
  • σ trend monitoring — the paper found σ is the leading indicator. The tool tracks it across turns.
  • Bell curve visualization — the paper showed that bimodal distributions signal contamination. The tool renders the histogram + Gaussian fit + rug plot in real time.
  • Per-chunk rug plot — the paper showed that individual chunk scores tell the eviction story. The tool plots every measurement.

The tool is the research, productized.

Context Inspector vs MCP Inspector

Anthropic MCP Inspector

Manual debugging

Great for verifying that MCP tools return correct data. Manual testing of individual tool calls.

  • Tool input/output inspection
  • Error messages, latency
  • Schema validation
  • One request at a time

Answers: "Is my tool call working?"

Context Inspector

Proactive monitoring

Watches the statistical health of the context window across turns. Alerts before output degrades.

  • Domain alignment bell curve
  • σ trend detection
  • Per-chunk scoring + rug plot
  • Continuous monitoring across workflow

Answers: "Is my context about to fail?"

They're complementary. MCP Inspector verifies tools work. Context Inspector verifies the context built from tool results remains structurally sound.

Three ways to use it

MCP server for AI workflows. CLI for scripts. Web dashboard for exploration.

MCP Server

npx contrarianai-context-inspector --setup

AI-guided wizard scans your project, suggests config, finds structural issues, generates .mcp.json. The easiest way to start.

CLI

npx contrarianai-context-inspector file.txt --verbose

Reports σ, mean, histogram, per-chunk breakdown. Pipe JSON output to jq or store for comparison. Reads from stdin.

Web Dashboard

Interactive bell curve with rug plot, ±1σ/2σ bands, Gaussian fit. Concentrator toggle. Chunk size slider. Real-time analysis.
