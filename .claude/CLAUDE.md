# CLAUDE.md — contrarianAI

## Project Overview

contrarianAI sells AI audit and analysis tools and services. The business thesis: most organizations building with AI agents are making predictable, documented mistakes — and paying for them in failed deployments, token waste, trust erosion, and silent failures. contrarianAI packages diagnostic expertise into repeatable audits, tooling, and advisory services.

## Core Positioning

The name "contrarianAI" reflects the approach: challenging the default assumptions that cause AI projects to fail. Not selling hype — selling the unglamorous engineering that makes AI actually work in production.

Key contrarian positions (sourced from research):
- A smarter model doesn't fix agent failures; a smarter environment does
- 90% of automation needs are workflows, not agents ("workflows wearing agent costumes")
- The model was never the problem — it's a meaning problem
- Agents cannot reliably judge their own output
- More data degrades AI performance when context isn't managed
- Type-checking and complexity impose measurable costs on AI-assisted development
